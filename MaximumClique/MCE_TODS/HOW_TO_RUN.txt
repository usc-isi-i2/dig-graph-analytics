Running Cliques for sellers, phones and emails for 2 months data

1. First run a spark job generate-data.py which outputs
S1 -> P1
S1 -> P2
S1 -> E1
P1 -> S1,E1
E1 -> S1,P1
etc in strings

spark-submit generate-data.py $GITHUB_HOME/dig-graphx/datasets/framer-text/Seller

2. Now run another python script str_to_int.py to map these strings to integers ranging from 0 to MAX-1
1234
0 1
2 1
3 4
python str_to_int.py intermediate/part-0000 clique-input
This also stores a dictionary clique-input_dict for string to integer mapping.


3. Now run edge to get the format required for MAxClique algorithm
./edge clique-input 
This will generate clique-input_new.txt

4. Now run MCE code 

./mce clique-input_new.txt 1024 1000 10 1

5. For postprocessing

postprocess.py clique-input_new.txt.cliquetree.txt clique-input_rev-dict clique-postprocessed 5

<input> - cliques
<rev-dict>  - with integer to string mapping
<output> - outputFile





For optimization,
run
g++ edge2standard.cpp -o3 -o edge
